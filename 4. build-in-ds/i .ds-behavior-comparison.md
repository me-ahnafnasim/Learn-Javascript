### JavaScript Data Structures Comparison: Array, Object, Set, Map, **String**

| Feature                   | Array                                   | Object                                                                          | Set                                           | Map                                                              | **String**                                                  |
|--------------------------|----------------------------------------|----------------------------------------------------------------------------------|----------------------------------------------|------------------------------------------------------------------|------------------------------------------------------------|
| **Ordered**              | Yes (insertion order)                  | Mostly (property order is defined in modern JS, but don’t rely on it for logic)  | Yes (insertion order)                        | Yes (insertion order)                                            | **Yes (character sequence)**                               |
| **Mutable**              | Yes                                    | Yes                                                                              | Yes                                          | Yes                                                              | **No (immutable)**                                         |
| **Allows Duplicates**    | Yes                                    | Keys: No, Values: Yes                                                            | No (unique values only)                      | Keys: No, Values: Yes                                            | **Yes (duplicate characters allowed)**                     |
| **Indexing / Access**    | Yes (by numeric index)                 | Yes (by key: string/symbol)                                                      | No direct indexing; use `has()`              | Yes (by key of any type)                                         | **Yes (by numeric index; read-only)**                      |
| **Syntax**               | `[1, 2, 3]`                            | `{ a: 1, b: 2 }`                                                                 | `new Set([1, 2, 3])`                         | `new Map([["a", 1], ["b", 2]])`                                  | `"text"`, `'text'`, or <code>\`text\`</code>              |
| **Use Case**             | Ordered lists, collections, iteration   | Simple key-value storage, JSON-like data                                         | Unique items, fast membership checks         | Key-value pairs with any key type, frequent add/remove/lookups   | **Text representation, identifiers, serialization**        |
| **Example**              | `const fruits = ["apple", "banana"];`  | `const person = { name: "Alice", age: 25 };`                                     | `const uniqueNums = new Set([1, 2, 2, 3]);`  | `const ages = new Map([["Alice", 25]]);`                         | `const msg = "hello";`                                    |
| **Can Store Any Type**   | Yes                                    | Keys: string/symbol; Values: any                                                 | Yes (any values, uniqueness by SameValueZero)| Keys: any type; Values: any                                      | **No — only UTF-16 characters (text)**                    |
| **Performance (typical)**| Fast iteration; lookup by value is O(n) | Fast property access; may have edge cases with prototype keys                    | Fast `has/add/delete` (avg O(1))             | Fast `get/set/has/delete` (avg O(1)); better for lots of entries | **O(1) char access; operations like concat/create new string (O(n))** |
